About Me

I've found that my best work comes from the projects I'm most passionate about. When I'm fully engaged in a problem, I'm driven to find the most elegant and efficient solution.

I began programming in high school through FIRST Robotics, where I helped design autonomous systems for our competition robots. Our team earned the Innovation in Control and Autonomous awards for work that fused Java, sensor feedback, and PIDF tuning into precise motion control. My proudest achievement wasn’t a trophy—it was reducing autonomous path error by analyzing log data, optimizing Bezier-curve velocity profiles, and characterizing each robot’s friction and voltage behavior until our trajectories ran flawlessly. That early focus on optimization and systems analysis still shapes how I engineer software today.

Since then, I’ve evolved from robotics to full-stack and embedded development, building end-to-end solutions across API, database, and front-end layers. Projects like PantryPal, where I re-architected ChatGPT’s response delivery into streamed, incremental updates for low-latency user experience, and Jack Donuts Rewards App, a Firebase-based loyalty platform for small businesses, strengthened my ability to design scalable, maintainable systems that balance reliability with usability.

At UC San Diego, I served as a Software Engineering Tutor, collaborating across 40+ student codebases to debug Android MVC architectures, construct user stories, and help teams meet deliverables under Agile sprints. The experience reinforced my collaborative instincts and ability to translate complex problems into actionable solutions.

Most recently, as a Tech Camp Instructor for underrepresented high-school students, I led workshops on AI, cybersecurity, and game design—showing new developers that creativity and logic can coexist.

I’m based in Lynwood, California, and approach engineering like an optimization problem: define constraints, iterate relentlessly, and refine until it’s elegant, efficient, and scalable. I thrive on turning late-night curiosity into production-ready systems. I exhaust every resource, ask questions fearlessly, and never stop improving the process behind the product.

Real-Time Ray Casting Engine

A pseudo-3D ray-casting demo that projects a 2D grid into a first-person view, complete with dynamic lighting. Click to learn more…

Three.js WebGL JavaScript Physics
View Code Live Demo

This project adapts classical 2D ray-casting techniques from Lode's Raycasting Tutorial to create a pseudo-3D environment. Rays are cast from a camera through a 2D map, and walls are projected onto the screen to simulate depth. Dynamic lighting further enhances the illusion of three-dimensional space—without requiring a full physics engine. Future updates may incorporate more advanced features like volumetric effects, interactive objects, or actual 3D physics. 

Algorithm RayCast2DTo3D(grid, camera):
    For each column x in the screen width:
        1. Calculate the angle of the ray based on camera direction & x.
        2. Step through the 2D grid until hitting a wall or max distance.
        3. Compute distance to wall & project onto the screen:
                - The closer the wall, the taller the column of pixels.
        4. Apply lighting or shading based on distance or angle.
        5. Render the column in the final frame buffer.

Algorithm DFS(start, goal):
    frontier ← stack with start
    explored ← empty set

    while frontier is not empty:
        current ← frontier.pop()       // LIFO order
        if current equals goal:
            return path from start to current

        mark current as visited (add to explored)
        for each neighbor of current:
            if neighbor is within grid bounds 
            and not an obstacle and not visited:
                push neighbor to frontier
                record parent of neighbor as current

    return failure (no path found)
                          

Algorithm BFS(start, goal):
    frontier ← queue with start
    explored ← empty set

    while frontier is not empty:
        current ← frontier.dequeue()   // FIFO order
        if current equals goal:
            return path from start to current

        mark current as visited (add to explored)
        for each neighbor of current:
            if neighbor is within grid bounds 
            and not an obstacle and not visited:
                enqueue neighbor to frontier
                record parent of neighbor as current

    return failure (no path found)
                          

Algorithm UCS(start, goal):
    frontier ← priority queue with (cost=0, start)
    explored ← empty set

    while frontier is not empty:
        (current_cost, current) ← frontier.pop()  // lowest cost first
        if current equals goal:
            return path from start to current w/ total cost current_cost

        mark current as visited (add to explored)
        for each neighbor of current:
            if neighbor is within grid bounds 
            and not an obstacle and not visited:
                new_cost ← current_cost + cost(neighbor)
                add (new_cost, neighbor) to frontier
                record parent of neighbor as current

    return failure (no path found)
                          
Algorithm A*(start, goal):
    Define h(n) as the heuristic
    frontier ← priority queue with (f = h(start), g = 0, start)
    explored ← empty set

    while frontier is not empty:
        // pop lowest f first
        (current_f, current_g, current) ← frontier.pop() 
        if current equals goal:
            return path from start to current with total cost current_g

        mark current as visited (add to explored)
        for each neighbor of current:
            if neighbor is within grid bounds 
            and not an obstacle and not visited:
                new_g ← current_g + cost(neighbor)
                new_f ← new_g + h(neighbor)
                add (new_f, new_g, neighbor) to frontier
                record parent of neighbor as current

    return failure (no path found)
